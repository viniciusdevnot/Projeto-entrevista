Quais os tipos de testes?

O teste automatizado é a aplicação de ferramentas de software para automatizar um processo manual conduzido por humanos de revisão e validação de um produto de software.

Testes Unitarios

São testes que verificam se uma parte específica do código, costumeiramente a nível de função, está funcionando corretamente

Saber Mais

Testes end-to-end

Os testes end-to-end são utilizados no processo de desenvolvimento de software para testar a função e os níveis de desempenho de uma aplicação quando utilizada como um produto.

Saber Mais

mock test

São normalmente criados para testar o comportamento de outros objetos. Em outras palavras, os objetos mock são objetos “falsos” que simulam o comportamento de uma classe ou objeto “real” para que possamos focar o teste na unidade a ser testada.

Saber Mais

Testes de interface

O teste da interface do utilizador (IU), por vezes conhecido como teste GUI dependendo do contexto, é uma série de acções utilizadas para medir o desempenho e a funcionalidade global dos elementos visuais de uma aplicação.

Testes de cenário

É possível incluir cenários de teste em testes de unidade para teste automatizado usando o método testDriver.runTestScenario().
